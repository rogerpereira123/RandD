Developer Guidelines

Getting Started

Gaana.com is a commercial music streaming service providing free and licensed music. Gaana Apps provides both Indian and International music tracks. Gaana has released its api's which made it asy for developers to discover and experience music in his own applications. At the heart of the Music API is an extensive database of information about albums. The Music API expands on Album data with collections of Song, Release, Composition, and Performance. Taken together, these collections of data provide entertainment information that will satisfy everyone from the casual consumer to the most avid music connoisseur.

Authentication

The API uses three different levels of authentication, depending on the method.
  • None: No authentication. Anybody can query the method.
  • API key: Requires an API key.
      Example: api_key=PyezS3Q4Smivb24d9SzZGYSuh--IaMfAkE
  • Login Authentication: Requires sign in process to complete before query the api.

    • Creating an App

    Developing an app is very easy. First, create the gaana folder, which is the workspace where you place your apps:

  • htdocs/gaana(Windows)


    • The basic files are:
  • index.html: This is required in all applications, and is the first page that will be displayed by Gaana when your application is loaded.
  • app.js: This is the main javascript file to get JSON data from API and responsible to diplay them on HTMl page.
  • get.php: This is the main PHP page to hit Gaana API's and request for play lists and songs
  • constant.php: This page contains constants variable which are used globally in other php files like BASE_URL
  • app.css: This page contains all styles which are used in our application for better look and feel.

    • Other files:
  • jquery-1.7.2.min.js: This is the main js file which allowed us to use jquery functions to do javascript programming.
  • jsPlayer.js: Including this file we are able to play songs using players.
  • jquery-ui.css: jQuery UI includes a robust CSS Framework designed for building custom jQuery widgets.
  • GetClass.php: This is the supporting file for sending get request to Gaana API
  • GetTracks.php: This is the supporting file of GetClass.php to process and load data of Songs category.
  • GetAlbums.php: This is the supporting file of GetClass.php to process and load data of Albums category.
  • GetArtists.php: This is the supporting file of GetClass.php to process and load data of Artists category.
  • GetPlaylists.php: This is the supporting file of GetClass.php to process and load data of Playlist category.

    • Gaana URIs

    http://s.staging.api.gaana.com
    This URI is used to get songs details and load them in our application. We used one constant variable for above url "SERVICEBASEURL".

    • Parameters

  • type: This is the main parameter which contains values like songs, artist, albums, playlist. Its value filters the reponse result based on types of songs, like so:
    http://s.staging.api.gaana.com/type=playlist
  • subtype: Subtype is the parameter whose value sub categorize the API result response into most popular, hot songs and other sub categories, like so:
    http://s.staging.api.gaana.com/type=playlist&subtype=most_popular_playlist
  • token: This is the parameter which is if provided then API will return user specific information. This parameter is optional.
  • format: This parameter value could be json. This parameter is optional.
  • order: Data returned from the database will not be in ordered format untill we will not pass one parameter in query string for defining order of data. The order can be defined in 24hrs, week, month or alltime. By default it is set to week.
  • limit: All results are paginated and set to a limit of 20 per page by default. This can be set directly via a 'limit' query string parameter, bydefault it is 40, like so:
    • There are some more parameters other than the parameters passed to get data from Gaana database.

    Debugging

    You can use your browser developer tools to debug your web app. Please know that the web inspector will only show errors/warnings in HTML/JS land - more errors may lurk behind the scenes. Use the console log to find out if your application is violating the sandbox, exceptions are thrown, etc. If you are using the desktop client, open the Inspector. You can do it right-clicking and clicking on Show Inspector or opening the Develop > Show Inspector menu item. If you are developing on a browser, it is likely that it will have a console integrated in its Developer Tools. Please check the documentation for your particular browser.

    Performance

    As a web application, you can apply web performance optimization techniques to deliver the best experience to the users of your app:
  • Load CSS before Javascript files.
  • Write performant CSS and Javascript. For instance, use efficient CSS selectors and minimize DOM manipulation.
  • Try to make the most out of browser capabilities, being sensible when using libraries.
  • Try to minimize the size of your bundle by only including the needed resources.
  • Optimize the bundled images.

    • We recommend you to use the developer tools that come built-in with browsers to measure performance.
